# DataOps Copilot

中文名：「AI 数据运营分析工作台」

DataOps Copilot 是一个用于面试展示的 AI + 数据分析 + 运营决策工作台。它面向数据分析、数据运营、产品运营、产品经理等岗位，重点展示从业务场景、指标体系、异常识别到 AI 报告生成的完整闭环。

## 为什么做这个项目

很多运营团队每天都在重复查数、做报表、找异常、写周报。这个项目把这些动作产品化，让面试官可以在 3 分钟内看到：

- 我能建立指标体系和指标口径。
- 我能用数据拆解经营、转化和用户反馈问题。
- 我能把异常识别、图表表达和运营建议落成一个真实工作台。
- 我不是只做聊天机器人外壳，而是把 AI 放进业务分析流程里。

## 面向岗位

- 数据分析师
- 数据运营
- 产品运营
- 增长运营
- 产品经理
- AI 产品/AI 运营相关岗位

## 功能模块

- 业务场景切换：经营分析、转化分析、用户反馈。
- KPI 指标区：展示当前值、环比、指标状态、指标口径。
- 趋势图：展示核心指标随时间变化。
- 维度对比图：按校区、渠道、反馈来源等维度拆解。
- 漏斗/排行图：展示转化漏斗或高频问题排序。
- 异常洞察：通过规则识别风险，并给出证据和建议。
- AI 报告预览：生成结构化经营分析报告。
- 指标口径与 SQL：每个核心指标都有定义和示例 SQL。
- 面试讲解模式：内置项目背景、技术亮点、业务亮点和 1 分钟介绍稿。

## 数据分析方法

项目内置三个 demo 业务场景。

### 经营分析

用于体现校园配送/本地生活运营经历。

核心指标：

- 订单量
- 收入
- 履约率
- 成本率
- 客诉率
- 渗透率

分析方法：

- 按当前周期和上一周期计算环比。
- 按校区拆解订单量、成本率、履约率和客诉率。
- 识别高成本区域、履约异常区域和客诉异常点。
- 输出运营建议，例如骑手供给、配送半径、商家出餐 SLA。

### 转化分析

用于体现数据分析和产品运营能力。

核心指标：

- 曝光量
- 线索转化率
- 成交转化率
- CAC
- 成交金额
- 试用成交率

分析方法：

- 构建曝光、线索、注册、试用、成交漏斗。
- 按渠道计算转化率和获客成本。
- 识别低转化、高 CAC 渠道。
- 输出渠道预算、素材测试和落地页优化建议。

### 用户反馈

用于体现产品运营/产品经理能力。

核心指标：

- 反馈量
- 负向反馈占比
- 平均处理时长
- 满意度
- 未解决问题
- 最高频问题

分析方法：

- 聚合反馈来源、问题类型、情绪、满意度和解决状态。
- 识别高频问题和负向反馈上升点。
- 根据处理时长和满意度判断产品体验风险。
- 输出产品优化建议和需求优先级。

## AI 模块设计

项目实现了 `AIProvider` 抽象，并提供 `MockAIProvider`：

- `AIProvider` 定义 AI 洞察摘要和报告建议的接口。
- `MockAIProvider` 返回稳定的本地 AI 文案，避免面试现场依赖网络和真实 API。
- `InsightEngine` 负责基于规则生成异常洞察。
- `ReportGenerator` 负责把指标、异常和 AI 建议组合成结构化分析报告。

后续接入真实大模型时，可以新增：

- `OpenAIProvider`
- `ClaudeProvider`

建议把真实 API 调用放到 Next.js Route Handler 或独立后端中，避免在浏览器暴露 API Key。

## 如何打开和运行

先说明一个容易踩坑的点：`http://localhost:3000` 只代表你本机正在运行的临时开发服务器。只要终端里的 `npm run dev` 停了，这个地址就会打不开；别人电脑也不能直接访问你的 `localhost`。

这个项目已经配置为静态导出模式。构建后会生成 `out/` 文件夹，它可以作为普通静态网页部署到 GitHub Pages、Vercel、Netlify，也可以在本地用静态预览服务打开。

### 本地开发

用于修改代码时实时预览：

```bash
npm install
npm run dev
```

浏览器打开：

```bash
http://localhost:3000
```

### 生成静态网页

```bash
npm run build
```

构建成功后会生成：

```bash
out/index.html
```

### 本地预览静态版本

用于模拟部署后的效果：

```bash
npm run preview
```

浏览器打开：

```bash
http://127.0.0.1:3000
```

如果 3000 端口被占用，可以指定端口：

```bash
npm run preview -- --port 4173
```

### 给别人电脑访问

推荐部署到 GitHub Pages 或 Vercel，得到一个公网 URL。这样面试官或其他电脑只需要打开链接，不需要安装 Node.js，也不需要运行本地项目。

#### GitHub Pages

1. 把项目推送到 GitHub 仓库的 `main` 分支。
2. 在仓库设置里开启 Pages，并选择 GitHub Actions 作为发布来源。
3. 本项目已包含 `.github/workflows/deploy-pages.yml`，推送后会自动执行：

```bash
npm ci
npm run build
```

4. GitHub Actions 完成后，会得到一个类似这样的地址：

```bash
https://你的用户名.github.io/仓库名/
```

#### Vercel

也可以把 GitHub 仓库导入 Vercel。Vercel 会自动识别 Next.js 项目并部署，适合快速生成稳定公开视频链接。

### 面试演示建议

- 首选：使用 GitHub Pages 或 Vercel 的公网链接，任何电脑都能打开。
- 备用：提前运行 `npm run build`，用 `npm run preview` 打开本地静态版本。
- 不建议只依赖 `npm run dev`，因为它是开发服务器，关掉终端后页面就无法访问。

## 面试讲解话术

这个项目叫 DataOps Copilot，是我为数据分析、数据运营和产品运营岗位准备的 AI 数据运营分析工作台。它不是普通聊天机器人，而是把运营分析流程做成一个可演示的产品闭环：选择场景后，系统会基于本地 mock 数据计算 KPI、趋势、维度对比和异常规则，再用 Mock AI 生成洞察解释和结构化报告。三个场景分别对应经营分析、转化分析和用户反馈，能展示我对指标口径、SQL 思维、异常识别、图表表达和运营决策落地的理解。技术上我用了 Next.js、TypeScript、Tailwind、Recharts，并抽象了 AIProvider，后续可以替换为 OpenAI 或 Claude API。
