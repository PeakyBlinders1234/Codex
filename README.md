# DataOps Copilot

中文名：「AI 数据运营分析工作台」

DataOps Copilot 是一个 AI 数据运营能力展示 Demo。它把业务数据、指标计算、异常识别、AI 洞察、经营报告和行动建议串成一个可操作的 Contemporary Data Studio，让访问者一打开网页就能看到 AI 能力如何落到真实运营分析流程里。

项目不接登录、数据库或后端服务，全部使用本地 mock 数据和稳定 Mock AI 输出，适合公开展示、作品集演示和离线预览。

## 为什么做这个项目

很多运营团队每天都要重复查数、做报表、找异常、写复盘、拆行动。这个项目把这些动作产品化，重点展示：

- 用指标体系承接经营、增长和用户反馈问题。
- 用规则引擎识别异常，而不是只展示静态图表。
- 用 AIProvider 抽象把洞察解释、报告生成和行动建议接入分析流程。
- 用 Slot Filling 补充询问机制解决自然语言问数里的条件缺失问题。
- 用指挥舱 UI 把“数据输入 -> 指标计算 -> 异常识别 -> AI 洞察 -> 报告生成 -> 行动建议”做成完整闭环。
- 让 AI 看起来不是聊天窗口外壳，而是一个能辅助业务决策的产品模块。

## 功能模块

- AI 运营指挥舱：首屏展示业务场景、经营健康分、3D 数据场景、风险队列和 AI 工作流。
- Contemporary Data Studio UI：默认浅色专业界面，采用左侧导航、顶部状态栏和 bento 工作区，保留更克制的深色演示模式。
- 轻量 3D 景深：使用 Three.js 实现粒子点阵和数据流曲线，作为辅助理解的数据空间层次，而不是纯装饰特效。
- AI Ask Bar：在侧边栏提供问数入口，一键进入 Slot Filling 演示流程。
- 业务场景切换：经营分析、转化分析、用户反馈三套 demo 数据。
- 自然语言问数：演示“我要查潍坊工程上周单量”时触发补充询问，要求补齐校区 ID 和明确时间范围。
- KPI 指标区：展示当前值、环比、状态、指标口径和 SQL。
- 趋势与对比图：用 Recharts 展示趋势变化、维度拆解、漏斗和排行。
- 异常洞察：基于阈值规则生成风险卡片，并给出证据和建议。
- AI 报告预览：生成结构化分析报告，包含摘要、关键发现、原因、建议和下一步动作。
- 行动任务看板：把洞察转成负责人、优先级、周期和预期影响。
- 业务星图总控台：用 Three.js 把指标、预警和行动映射成可交互关系网络。
- AI 能力展示：说明数据输入、指标建模、AI 洞察、报告生成和行动闭环。
- 双主题系统：默认浅色专业模式，也可切换深色演示模式。

## 数据分析方法

项目内置三个 demo 业务场景，切换场景后 KPI、图表、洞察、报告和行动任务会同步变化。

### 经营分析

用于展示本地生活/校园配送经营分析能力。

核心指标：

- 订单量
- 收入
- 履约率
- 成本率
- 客诉率
- 渗透率

分析方法：

- 按当前周期和上一周期计算环比。
- 按校区拆解订单量、成本率、履约率和客诉率。
- 识别高成本区域、履约异常区域和客诉异常点。
- 输出骑手供给、配送半径、商家出餐 SLA 等运营建议。

### 转化分析

用于展示增长漏斗和渠道优化能力。

核心指标：

- 曝光量
- 线索转化率
- 成交转化率
- CAC
- 成交金额
- 试用成交率

分析方法：

- 构建曝光、线索、注册、试用、成交漏斗。
- 按渠道计算转化率和获客成本。
- 识别低转化、高 CAC 渠道。
- 输出预算迁移、素材测试和落地页优化建议。

### 用户反馈

用于展示产品运营和用户体验分析能力。

核心指标：

- 反馈量
- 负向反馈占比
- 平均处理时长
- 满意度
- 未解决问题
- 最高频问题

分析方法：

- 聚合反馈来源、问题类型、情绪、满意度和解决状态。
- 识别高频问题、负向反馈上升点和处理时长风险。
- 根据满意度和未解决问题判断产品体验压力。
- 输出产品优化建议和需求优先级。

### 自然语言问数

用于展示飞书 Aily 数据查询机器人的补充询问机制。

演示流程：

- 用户提问：“我要查潍坊工程上周单量”。
- 系统识别到学校简称和查询指标，但缺少明确校区 ID 和日期范围。
- 系统反问：“请提供具体校区ID和时间范围。”
- 用户补充：“校区ID：123，2026-05-13-2026-05-19单量”。
- 系统查询本地 `mock_order_daily`，返回单量看板和 Excel 风格明细表。

内置演示数据：

- 学校：潍坊工程职业技术学院（南小区）
- 校区 ID：123
- 时间范围：2026-05-13 至 2026-05-19
- 指标：单量

## AI 模块设计

项目实现了 `AIProvider` 抽象，并提供 `MockAIProvider`：

- `AIProvider` 定义 AI 洞察摘要和报告建议接口。
- `MockAIProvider` 返回稳定的本地 AI 文案，保证公开演示不依赖网络、API Key 或真实模型额度。
- `InsightEngine` 负责基于规则生成异常洞察。
- `ReportGenerator` 负责把指标、异常和 AI 建议组合成结构化报告。
- `CommandCenter` 派生模块负责生成健康分、预警队列、预测、行动项和业务星图节点模型。
- `SlotFillingQuery` 负责解析自然语言问数，校验校区 ID、时间范围和指标槽位，缺失时生成补充询问。

后续接入真实大模型时，可以新增：

- `OpenAIProvider`
- `ClaudeProvider`
- 企业内部模型 Provider

建议把真实 API 调用放到 Next.js Route Handler 或独立后端中，避免在浏览器暴露 API Key。MockAIProvider 可以继续作为离线展示、测试和降级兜底。

## 如何打开和运行

先说明一个容易踩坑的点：`http://localhost:3000` 只代表你本机正在运行的临时开发服务器。只要终端里的 `npm run dev` 停了，这个地址就会打不开；别人电脑也不能直接访问你的 `localhost`。

这个项目已经配置为静态导出模式。构建后会生成 `out/` 文件夹，它可以作为普通静态网页部署到 GitHub Pages、Vercel、Netlify，也可以在本地用静态预览服务打开。

### 本地开发

用于修改代码时实时预览：

```bash
npm install
npm run dev
```

浏览器打开：

```bash
http://localhost:3000
```

### 生成静态网页

```bash
npm run build
```

构建成功后会生成：

```bash
out/index.html
```

### 本地预览静态版本

用于模拟部署后的效果：

```bash
npm run preview
```

浏览器打开：

```bash
http://127.0.0.1:3000
```

如果 3000 端口被占用，可以指定端口：

```bash
npm run preview -- --port 4173
```

### 给别人电脑访问

推荐部署到 GitHub Pages 或 Vercel，得到一个公网 URL。这样其他电脑只需要打开链接，不需要安装 Node.js，也不需要运行本地项目。

#### GitHub Pages

1. 把项目推送到 GitHub 仓库的 `main` 分支。
2. 在仓库设置里开启 Pages，并选择 GitHub Actions 作为发布来源。
3. 本项目已包含 `.github/workflows/deploy-pages.yml`，推送后会自动执行：

```bash
npm ci
npm run build
```

4. GitHub Actions 完成后，会得到一个类似这样的地址：

```bash
https://你的用户名.github.io/仓库名/
```

#### Vercel

也可以把 GitHub 仓库导入 Vercel。Vercel 会自动识别 Next.js 项目并部署，适合快速生成稳定公开视频链接。

可以这样介绍这个作品：

> DataOps Copilot 是一个 AI 数据运营指挥舱。它不是简单聊天机器人，而是把数据运营流程做成一个完整产品：选择业务场景后，系统会基于本地数据计算 KPI、趋势、维度对比和健康分，自动识别风险，再用 Mock AI 生成洞察解释、经营报告和行动建议。问数模块还演示了 Aily 数据查询机器人里的 Slot Filling：当用户只说“我要查潍坊工程上周单量”时，系统不会直接查，而是要求补充校区 ID 和明确时间范围；补齐后再返回看板和 Excel 表格。后续只要把 MockAIProvider 替换成 OpenAI、Claude 或企业模型，就可以接入真实 AI 分析能力。
